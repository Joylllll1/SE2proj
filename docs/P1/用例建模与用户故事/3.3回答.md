AI在异常流的分析中能够指出一些常见的异常情况，但是分析并不完整，缺失了很多的异常情况。

例如：
在核心用例1中，AI的回答如下：
1.若用户在 CAS 页面输入密码错误，CAS 页面提示“账号或密码错误”，流程终止。
2.若用户非 NJU 在校生（无有效学号），CAS 拒绝授权，系统提示“仅限南大在校师生使用”。
3.若该学生的哈希标识在树洞平台的“封禁名单”中，系统拦截登录并提示“您的账号已被封禁，解封时间：YYYY-MM-DD”。

AI的回答是正确的，但是不够完善。例如在这一核心用例中，人工分析阶段还提出了以下几点：
1.若用户尝试注册已存在账号，系统提示“账号已存在”，并引导用户直接登录。
2.若所有认证方式均不可用，系统提示服务暂时不可用并中止流程。 



AI生成的验证标准有一部分非常好，是具体可测试的；
如在发布匿名帖子这一用户故事中，AI提出了以下的验收标准：
1. 支持文本输入，最多2000字。
2. 支持上传最多4张图片，单张不超过5MB，自动压缩。
这两个验收标准都是具体且可测试的。
但是AI同时也生成了一些不够具体的内容。
如同样在发布匿名帖子这一用户故事中，AI提出了以下的验收标准：
3. 任何前端页面和抓包均无法逆向出真实发帖人的身份ID。
这句话说的就比较宽泛笼统，不够具体，难以测试。
