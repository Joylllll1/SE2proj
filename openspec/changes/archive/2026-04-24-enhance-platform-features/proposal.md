## Why

当前 NJU 树洞平台在活动管理、内容审核和用户交互方面存在多个体验问题：

1. **活动时间输入不友好**：当前是纯文本输入，用户需要手动输入日期和时间，格式不统一
2. **活动分类不够丰富**：缺少"答辩"、"校招"等重要校园活动分类
3. **优质内容曝光不足**：通过审核的活动海报无法在首页轮播展示，缺少内容推广渠道
4. **举报审核效率低**：管理员无法查看帖子详情，只能看到片段内容，难以做出准确判断
5. **收藏功能过于简单**：只有单一收藏列表，无法分类管理，用户收藏多后难以查找
6. **活动无法收藏**：用户无法收藏感兴趣的活动，导致后续查找不便

这些问题影响了平台的可用性和用户体验，亟待改进。

## What Changes

### 活动管理增强

- **日历时间选择器**：活动时间从文本输入改为日历选择器，支持日期和时间选择
- **时间格式统一展示**：活动时间统一显示为 "4.28 周一 14:00" 格式
- **活动分类扩展**：新增"答辩"、"校招"、"实习招聘"等分类
- **活动支持收藏**：用户可以收藏感兴趣的活动

### 内容管理增强

- **首页轮播管理**：管理员可以选择通过审核的活动海报在首页轮播展示
- **举报详情查看**：管理员可以查看被举报帖子的完整内容

### 收藏功能升级

- **收藏文件夹系统**：用户可以创建自定义收藏夹
- **收藏时选择文件夹**：收藏帖子/活动时选择要保存到的文件夹
- **"全部"文件夹**：默认文件夹，所有收藏的内容都会在其中显示
- **活动收藏支持**：活动可以被收藏到文件夹

## Capabilities

### New Capabilities

- `event-date-picker`: 活动日期时间选择器能力，支持日历选择和格式化展示
- `homepage-carousel`: 首页轮播管理能力，管理员选择活动海报展示
- `report-detail-view`: 举报详情查看能力，管理员查看帖子完整内容
- `collection-folders`: 收藏文件夹管理能力，用户创建和管理收藏夹
- `event-bookmarking`: 活动收藏能力，用户可以收藏活动

### Modified Capabilities

- `bookmarks`: 收藏能力需要扩展支持文件夹系统（当前为单一列表）

## Non-Goals (非目标)

本次改进不涉及以下内容：

- 后端 API 对接：当前仍使用前端 mock 数据
- 移动端适配：移动端导航暂不改进
- 收藏夹分享/协作：收藏夹仅限个人使用
- 活动编辑功能：通过审核后不支持修改活动信息
- 轮播图自动轮播：轮播图手动切换，暂不实现自动播放

## Impact

### 受影响的代码

- `frontend/src/components/ComposePage.jsx`：活动发布表单，添加日历选择器、扩展分类选项
- `frontend/src/components/AdminPage.jsx`：新增轮播管理面板、举报详情弹窗
- `frontend/src/components/HomePage.jsx`：轮播图组件支持管理员配置的活动海报
- `frontend/src/components/BookmarksPage.jsx`：收藏文件夹管理、收藏时选择文件夹
- `frontend/src/components/PostCard.jsx` / `ActivityCard`：收藏时显示文件夹选择弹窗
- `frontend/src/App.jsx`：收藏数据结构扩展，支持文件夹和活动收藏

### 数据持久化

- localStorage 新增键：
  - `nju_collection_folders`：用户收藏夹列表
  - `nju_bookmarks_with_folder`：带文件夹信息的收藏数据
  - `nju_carousel_items`：首页轮播配置

### 用户体验影响

- **活动发布者**：时间输入更准确，分类选择更丰富
- **管理员**：可以查看帖子详情做出更准确的审核判断，可以推广优质活动
- **普通用户**：收藏可以分类管理，活动可以收藏收藏，内容发现更便捷
