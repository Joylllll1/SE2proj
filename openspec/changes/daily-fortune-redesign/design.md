## Context

当前每日运势功能已实现基本功能，但存在以下问题：
1. 未打卡状态过于花哨，有神秘提示、背景装饰等，不符合洛谷的简洁风格
2. 宜忌内容偏向明显（"忌"多为贬义词如"熬夜"、"拖延"等）
3. 宜/忌只有简单词语，缺少有趣的详细描述

洛谷的设计理念：
- 未打卡：极简设计，仅显示日期
- 已打卡：清晰展示运势等级、宜忌（每项有简短描述）

## Goals / Non-Goals

**Goals:**
- 未打卡状态：模仿洛谷，仅显示日期，无额外装饰
- 宜忌内容：中性词语，避免明显偏向
- 每个宜/忌：包含有趣的具体描述（2-3句话）
- 数量调整：2宜 + 2忌
- 保持玻璃拟态整体风格

**Non-Goals:**
- 不改变随机算法
- 不改变localStorage存储逻辑
- 不移除历史记录功能

## Decisions

### 1. 未打卡状态设计
**决策**: 采用洛谷的三栏式日历布局，所有内容动态计算
**理由**: 洛谷的日历设计简洁优雅，符合用户期待

**布局结构**:
```
┌─────────────────────────────┐
│   欢迎回来，[用户名]         │  ← 用户信息区
│                             │
│  [农历]  [日期]  [星期]     │  ← 日历核心区（三栏）
│  (动态)  (动态)  (动态)      │
│                             │
│     [ 点击打卡 ]             │  ← 操作按钮（亮橙色）
└─────────────────────────────┘
```

**各部分规格**:
- **整体容器**: 玻璃拟态风格
  - 背景：`bg-white/60` 半透明白色
  - 边框：`border border-line-soft`
  - 效果：`backdrop-blur-md` 毛玻璃
  - 阴影：`shadow-glass`
- **用户信息区**: 顶部显示"欢迎回来，[用户名]"，用户名蓝色加粗
- **日历核心区**: 三栏式水平布局，**全部动态计算**
  - 左侧：农历月份（如"四月小"、"五月大"）- 根据公历日期实时计算
  - 中间：公历日期数字（如"25"）- 当前日期的"日"
  - 右侧：星期（如"星期六"）- 当前星期几
- **操作按钮**: 亮橙色渐变背景，白色文字"点击打卡"，圆角长方形

**动态计算规则（关键）**:
- 左侧农历：使用农历转换算法，根据当前公历日期计算农历月份和大小月
- 中间日期：`new Date().getDate()` 获取当前日期
- 右侧星期：`new Date().getDay()` 获取当前星期（0-6，转换为中文）
- 所有内容每天自动更新，无需人工维护

### 2. 宜忌内容策略
**决策**: 使用中性活动词语，添加生活化描述
**理由**: 避免价值判断，让描述本身有趣

**中性词语原则**:
- "宜"和"忌"使用相同类型的活动词语
- 避免明显的贬义词（如"拖延"、"懒惰"等）
- 避免明显的褒义词（如"努力"、"勤奋"等）
- 使用中性活动描述（如"晨跑"、"阅读"、"外卖"、"咖啡"等）

**描述风格区分（关键）**:
- **宜的描述**：鼓励、积极、推荐 → "适合做X，因为..."
- **忌的描述**：轻松提醒、幽默 → "可以X，但是..." 或 "偶尔X也不错"

**格式**:
```
宜：[活动]
[鼓励式描述]

忌：[活动]
[轻松提醒式描述]
```

**示例对比**:

❌ **旧版本（有偏向）**:
```
宜：写作业
学习让人进步，今天要加油！

忌：熬夜
熬夜伤身体，早点睡吧！ ← 说教语气
```

✅ **新版本（中性 + 区分描述）**:
```
宜：晨跑锻炼
清晨的空气最新鲜，跑完神清气爽 ← 鼓励推荐

忌：点外卖
偶尔犒劳自己，外卖也很香 ← 轻松提醒
```

**内容库示例（中性词语 + 描述风格区分）**:

**宜的选项**（鼓励推荐式）:
- 晨跑锻炼 → "清晨的空气最新鲜，跑完神清气爽"
- 阅读经典 → "沉浸书中的世界，让思绪慢慢沉淀"
- 整理书桌 → "整洁的环境让心情也变得清爽"
- 学习烹饪 → "自己做的饭菜，总是格外香"
- 练习书法 → "一笔一划间，心也慢慢静下来"
- 早起早睡 → "规律作息让你精力充沛"
- 健身运动 → "出汗的感觉真好，身体更轻松"

**忌的选项**（轻松提醒式）:
- 点外卖 → "偶尔犒劳自己，外卖也很香"
- 喝咖啡 → "下午茶时光，一杯咖啡刚刚好"
- 刷短视频 → "轻松一下，刷几个有趣的视频"
- 睡懒觉 → "周末的早晨，多睡一会儿也无妨"
- 玩游戏 → "游戏也是一种放松方式，适度就好"
- 熬夜追剧 → "剧情太精彩，但也要注意休息哦"
- 吃甜食 → "甜甜的美食让人心情变好"

### 3. 宜忌描述风格
**决策**: 每个宜/忌包含：
- 主体活动（4-6字）
- 描述（1-2句话，轻松幽默）

**格式**:
```
宜：[活动]
[有趣描述]
```

### 4. 数量调整
**决策**: 2宜 + 2忌
**理由**: 洛谷风格简洁，太多内容显得冗余

## 数据结构设计

### 新的宜忌数据结构
```typescript
interface FortuneItem {
  activity: string;    // 活动名称（4-6字）
  description: string; // 有趣描述（1-2句话）
}

interface FortuneContent {
  fortunes: {
    type: string;
    text: string;
  }[];
  dos: FortuneItem[];      // 今日宜（2个）
  donts: FortuneItem[];    // 今日忌（2个）
}
```

### 内容示例
```typescript
{
  dos: [
    { activity: "晨跑锻炼", description: "清晨的空气最新鲜，跑完神清气爽" },
    { activity: "阅读经典", description: "沉浸书中的世界，让思绪慢慢沉淀" }
  ],
  donts: [
    { activity: "暴饮暴食", description: "胃也是需要休息的，七分饱刚刚好" },
    { activity: "熬夜刷剧", description: "剧情可以明天再看，皮肤不能等明天" }
  ]
}
```
